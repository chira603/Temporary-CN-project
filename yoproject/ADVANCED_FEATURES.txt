‚úÖ FEATURE 2: DNSSEC Validation Path Visualization
---------------------------------------------------

Backend Implementation:
- Complete DNSSEC chain simulation with 7 detailed steps:
  1. Root DNSKEY Retrieval (trust anchor)
  2. Root DS Record Verification for TLD
  3. TLD DNSKEY Retrieval
  4. TLD DS Record Verification for domain
  5. Authoritative DNSKEY Retrieval
  6. RRSIG Signature Verification
  7. Final Validation Summary

- Each step includes:
  * Level (root/tld/authoritative)
  * Record type (DNSKEY/DS/RRSIG)
  * Key tag (unique identifier)
  * Algorithm (RSASHA256)
  * Digest type (SHA256)
  * Signature expiration/inception dates
  * Validation status (valid/invalid)
  * Days until expiration

- DNSSEC Failure Simulation:
  * Configurable via "Simulate DNSSEC Failure" checkbox
  * Simulates DS digest mismatch at root level
  * Shows security risk level
  * Provides detailed error messages

Frontend Components:
